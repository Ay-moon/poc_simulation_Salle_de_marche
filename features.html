Feature 1

Option Capture + Pricing Black-Scholes

User story : le trader saisit une option Call sur TotalEnergies → l'outil price avec BS → sauvegarde les Greeks → met à jour le blotter.

Règles métier

  • Tous les champs obligatoires : Underlying, Type, Strike, Maturity, Notional, BuySell
  • Strike > 0 (validation CPositionValidator)
  • Maturity > Today (une option expirée n'a pas de sens)
  • Notional doit être un multiple du lot (100 pour les listées)
  • Vol implicite récupérée depuis la surface de vol (K/S, T)
  • Modèle EU → Black-Scholes, modèle US → Binomial CRR
  • Prix et Greeks sauvegardés dans tbl_Greeks (SQL/Access)

Séquence multi-stack

sequenceDiagram participant UI as DealTicket (VBA) participant APP as CApp participant VS as VolSurface (VB.net COM) participant BS as BlackScholesEngine (C# COM) participant DB as SQL/Access UI->>APP: btnPriceOption_Click() APP->>VS: GetVol(spot=58.40, K=60, T=0.25) VS->>APP: impliedVol = 28% APP->>BS: Price(58.40, 60, 0.035, 0.28, 0.25, "CALL") BS->>APP: price=2.45€ APP->>BS: Greeks(58.40, 60, 0.035, 0.28, 0.25, "CALL") BS->>APP: [Δ=0.52, Γ=0.08, ν=0.18, θ=-0.02, ρ=0.05] APP->>DB: INSERT tbl_Greeks (all values) APP->>UI: RefreshBlotter() + UpdateGreeksDashboard()
VBADealTicket_Sheet.bas — Bouton Pricer
Private Sub btnPriceOption_Click()
    Dim app As CApp  : Set app = ThisWorkbook.App
    Dim ws  As Worksheet : Set ws = Me

    ' 1. Construire COption depuis les cellules UI
    Dim opt As New COption
    opt.Underlying = ws.Range("D5").Value    ' ex: "TTE.PA"
    opt.OptionType  = ws.Range("D6").Value    ' "CALL"
    opt.ExerciseStyle = "EU"
    opt.Strike      = ws.Range("D7").Value    ' 60.00
    opt.Maturity    = ws.Range("D8").Value    ' date
    opt.Notional    = ws.Range("D9").Value    ' 100000
    opt.BuySell     = ws.Range("D10").Value   ' "BUY"

    ' 2. Récupérer vol implicite depuis la surface (VB.net)
    Dim spot As Double
    spot = app.MarketData.GetSpot(opt.Underlying)

    Dim vol As Double
    vol = app.VolSurface.GetVol(spot, opt.Strike, opt.Maturity_Years)

    ' 3. Pricing via C# COM DLL
    Dim r As Double : r = app.Config.GetParam("DEFAULT_RISK_FREE")
    Dim price As Double
    price = app.PricingEngine.Price(spot, opt.Strike, r, vol, _
                                      opt.Maturity_Years, opt.OptionType)

    Dim greeks As Variant
    greeks = app.PricingEngine.Greeks(spot, opt.Strike, r, vol, _
                                        opt.Maturity_Years, opt.OptionType)

    ' 4. Sauvegarder et rafraîchir
    app.Workflow.Transition opt, "PRICÉ", price, greeks
    app.RefreshBlotter
    MsgBox "Option pricée : " & Format(price, "0.0000") & "€" & vbNewLine & _
           "Δ=" & Format(greeks(0),"0.000") & _
           " Γ=" & Format(greeks(1),"0.000"), vbInformation
End Sub

Feature 2

Dashboard Greeks — Agrégation par Book

User story : le Risk Manager voit les Greeks agrégés de tout le book en temps réel et reçoit une alerte si une limite est franchie.

Δ

Delta agrégé

Somme des Deltas × Notionaux × Spot de chaque position du book.

Exemple book EQUITY-01 :
TTE Call : +0.52 × 100K × 58.40 = +3 037K€
BNP Put : −0.38 × 50K × 42.10 = −800K€
Δ_book = +2 237K€
Limite : ±5 000K€ → OK ✅

Γ

Gamma agrégé

Gamma mesure la convexité du book en cash (€ par 1€ de variation spot).

Γ_book = +12 500€ /€²
Un Gamma positif = le book bénéficie des grands mouvements (long gamma). Un Gamma négatif = le book perd lors des grands mouvements (short gamma = risque de crash).
Limite Gamma book : ±25 000€/€²

ν

Vega agrégé

Sensibilité totale du book à 1% de hausse de la vol implicite.

ν_book = +48 000€ / 1%
Long Vega = le book gagne quand la vol monte (crise). Short Vega = le book perd en cas d'explosion de vol (risque vendeur d'options).
Limite Vega : ±100 000€/1%

Alerte limite automatique : CRiskService.CheckGreekLimits(bookId) est appelé après chaque reprice. Si un Greek dépasse la limite définie dans tbl_RiskLimits, une alerte rouge apparaît dans le dashboard Excel et une ligne est écrite dans tbl_AuditLog. Le Risk Manager reçoit une notification par email (Outlook Automation).

Feature 4

VaR Calculation — 3 Méthodes

User story : le Risk Manager lance le calcul VaR EOD et obtient les 3 méthodes avec comparaison et résultat de backtesting.

🎲
Quelle VaR utiliser ?
La VaR paramétrique est rapide mais ignore les non-linéarités (options). La VaR historique capture mieux les fat tails mais dépend de l'historique disponible. La VaR Monte Carlo est la plus précise pour les options mais est plus lente. On utilise les 3 et on retient la plus conservatrice.

Stress Testing — Scénarios de crise

User story : le Risk Manager simule l'impact d'un krach boursier ou d'une explosion de vol sur le book pour le Comité de Risque.

Pythonstress_tester.py
def run_stress(book_id: str, scenarios: list) -> dict:
    """Applique des scénarios de stress et
    retourne le P&L par scénario."""
    greeks = load_greeks(book_id)
    results = {}

    for scenario in scenarios:
        spot_shock = scenario['spot_shock']   # ex: -0.22
        vol_shock  = scenario['vol_shock']    # ex: +0.40

        # P&L delta + gamma (choc spot)
        pnl_spot = (greeks.Delta * spot_shock * greeks.Spot
                  + 0.5 * greeks.Gamma * (spot_shock * greeks.Spot)**2).sum()

        # P&L vega (choc vol)
        pnl_vol = (greeks.Vega * vol_shock * 100).sum()

        # Reprice complet optionnel (MC)
        results[scenario['name']] = {
            'pnl_spot': round(pnl_spot, 2),
            'pnl_vol':  round(pnl_vol, 2),
            'pnl_total': round(pnl_spot + pnl_vol, 2)
        }
    return results

Pour chaque nouvelle feature quant, suivre ce cycle de 8 étapes.

🔄

États et transitions impliqués. Séquence inter-langages (VBA → C# → Python → SQL). Cas d'erreur.

🏗️

💻

5. Implémentation

⚡

7. Perf Pass

Batch arrays pour COM. Parallel.For en C#. NumPy vectorisé en Python. Batch INSERT SQL. Timer avant/après.
