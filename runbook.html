💡 Expert Data Analyst parle au junior :
Tu te demandes pourquoi on enlève Access ? Imagine que tu as un cahier de notes (Access) vs un vrai dossier de classement dans une armoire verrouillée (SQL Server). Le cahier, si deux personnes écrivent en même temps, ça fout le bazar. L'armoire SQL Server, elle gère 100 personnes à la fois, elle garde une trace de tout, et si quelqu'un fait une bêtise, elle peut tout annuler. En salle de marché, une mauvaise donnée = une mauvaise décision = des millions perdus.

Installation

Mise en place en 8 étapes

1. Installer Python + dépendances

Exécuter setup_python_env.bat depuis le dossier CommandoQuant. Vérifier que xlwings addin install s'est bien déroulé (add-in visible dans Excel → Options → Compléments).

2. Compiler et enregistrer les DLLs .NET

Ouvrir CommandoQuant.sln dans Visual Studio. Build → Release. Exécuter register_dlls.bat en tant qu'Administrateur. Vérifier : CreateObject("CommandoQuant.BlackScholesEngine") dans la Immediate Window VBA ne doit pas lever d'erreur.

3. Vérifier les tables SQL Server existantes

Les tables existent déjà sur PERSO-AJE-DELL\BFASERVER01 / CommandoQuant. Vérifier dans SSMS : SELECT TABLE_NAME FROM INFORMATION_SCHEMA.TABLES WHERE TABLE_TYPE='BASE TABLE'. Doit retourner au moins : tbl_Positions, tbl_Greeks, tbl_VaR, tbl_MarketData, tbl_VolSurface, tbl_StressTest, tbl_AuditLog. Si tables manquantes, exécuter le DDL de la page Data Model.

💡 Junior : SSMS = SQL Server Management Studio. C'est le "tableau de bord" de SQL Server. Comme le Panneau de Contrôle de Windows, mais pour les bases de données. Tu peux y voir toutes les tables, lancer des requêtes, vérifier que tout va bien.

4. Ouvrir CommandoQuant.xlsm

Activer les macros. Ajouter la référence COM : VBE → Outils → Références → cocher "Microsoft Scripting Runtime" et "Microsoft ActiveX Data Objects 6.1". Activer l'add-in xlwings.

5. Configurer tbl_Config dans SQL Server

Dans SSMS, sur PERSO-AJE-DELL\BFASERVER01, mettre à jour les paramètres clés :

UPDATE tbl_Config SET ParamValue = 'PERSO-AJE-DELL\BFASERVER01' WHERE ParamKey = 'SQL_SERVER';
UPDATE tbl_Config SET ParamValue = 'CommandoQuant' WHERE ParamKey = 'SQL_DATABASE';
UPDATE tbl_Config SET ParamValue = '0.035' WHERE ParamKey = 'DEFAULT_RISK_FREE';
UPDATE tbl_Config SET ParamValue = '10000' WHERE ParamKey = 'MC_SIMULATIONS';
Feuille Excel "Config" → la chaîne de connexion SQL Server est désormais la seule configurée. Supprimer toute référence à Access.

6. Charger la surface de volatilité

Feuille "Vol Surface" → importer les données de vol implicite (format CSV Bloomberg ou saisie manuelle). Vérifier : ?ThisWorkbook.App.VolSurface.GetVol(58.4, 60, 0.25) retourne une valeur entre 0.15 et 0.50.

7. Charger l'historique de cours

Feuille "Market Data" → importer 250j minimum de cours de clôture et rendements pour chaque sous-jacent. La VaR historique nécessite au moins 250 observations.

8. Test de bout en bout

Saisir une position test : Deal Ticket → TTE.PA · Call · K=60 · T=3M · N=100 · BUY. Cliquer "Pricer" → vérifier le prix et les Greeks. Cliquer "Calc VaR" → vérifier les 3 résultats VaR. Contrôler tbl_AuditLog.

Troubleshooting

Problèmes fréquents & solutions

ErreurCause probableSolution
CreateObject("CommandoQuant.BlackScholesEngine") → Erreur 429 DLL C# non enregistrée comme serveur COM Exécuter register_dlls.bat en Admin. Vérifier que la DLL est bien en Release (pas Debug). Vérifier que le .NET Framework 4.8 est installé.
xlwings.call(...) → ModuleNotFoundError xlwings add-in non installé ou mauvais chemin Python Vérifier xlwings addin install. Dans Excel → Options xlwings → configurer le chemin du venv Python. Redémarrer Excel.
ADODB.Connection → Login failed for user Authentification SQL Server échouée (compte Windows sans accès) Dans SSMS sur PERSO-AJE-DELL\BFASERVER01 : CREATE LOGIN [DOMAINE\votrecompte] FROM WINDOWS; puis USE CommandoQuant; CREATE USER [DOMAINE\votrecompte] FOR LOGIN [DOMAINE\votrecompte]; ALTER ROLE db_datareader ADD MEMBER [DOMAINE\votrecompte];
pyodbc — Data source name not found Driver ODBC SQL Server non installé Télécharger "Microsoft ODBC Driver 17 for SQL Server" sur Microsoft. Relancer Python après installation.
Named Pipes Provider — could not open a connection Service SQL Server arrêté ou nom de serveur incorrect Vérifier dans services.msc que "SQL Server (BFASERVER01)" est Démarré. Vérifier le nom exact : PERSO-AJE-DELL\BFASERVER01. Activer TCP/IP dans SQL Server Configuration Manager.
Toute erreur mentionnant ".accdb" ou "Access Driver" Code non migré — référence à Access residuelle Chercher dans tout le code VBA et Python : "accdb", "Access Driver", "DBQ=", "Microsoft Access". Ces chaînes ne doivent plus exister. Remplacer par la chaîne SQL Server.
VolSurface.GetVol → Valeur 0 ou erreur Surface de vol vide ou non chargée Vérifier que la feuille "Vol Surface" contient des données. Relancer CApp.Initialize() pour recharger la surface. Vérifier le format des colonnes (Moneyness décimal, Maturity en années).
VaR Python retourne 0 pour toutes méthodes Historique de cours manquant dans tbl_MarketData Charger au minimum 260 jours de cours pour chaque sous-jacent. Vérifier que la colonne DailyReturn est calculée (non nulle). Tester avec python var_calculator.py --test.
Greeks tous à 0 dans le dashboard tbl_Greeks vide ou jointure erronée Vérifier qu'une position a bien été pricée. Dans SSMS sur PERSO-AJE-DELL\BFASERVER01 : SELECT TOP 5 * FROM tbl_Greeks ORDER BY CalcDate DESC. Le bouton "Pricer" doit insérer une ligne dans tbl_Greeks.
