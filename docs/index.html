Marché

Inspiré des systèmes quant professionnels

Les banques de financement et d'investissement (BNP Paribas CIB, Société Générale, Natixis, Crédit Agricole CIB) s'appuient sur des plateformes industrielles pour gérer le risque de marché sur actions. CommandoQuant en capture l'essence dans un stack accessible.

🔷

Murex MX.3

Référence mondiale pour les dérivés actions. Pricing multi-modèles (Black-Scholes, Dupire Local Vol, Heston stochastique), gestion des Greeks en temps réel, P&L attribution, VaR. CommandoQuant en reprend l'architecture en couches et les concepts : deal ticket, blotter, risk engine, reporting.

📊

Bloomberg MARS

Market Assisted Risk System de Bloomberg. Calcul de VaR paramétrique et historique, stress tests réglementaires (FRTB), surface de volatilité implicite. CommandoQuant s'inspire de ses workflows de calcul quotidien et de ses rapports réglementaires (EMIR, MiFID II).

⚙️

SunGard Adaptiv / MarkIT

Plateformes de risk management pour dérivés OTC. Gestion du cycle de vie des options, calcul des sensibilités, CVA/DVA. CommandoQuant reprend le pipeline de valorisation : market data feed → pricing engine → Greeks → VaR → rapport de risque.

Ce que CommandoQuant apporte : 100% open stack (Excel VBA + .NET + Python), zéro licence, déployable immédiatement, tout le code readable et évolutif. Idéal pour une mission commando : livrer un moteur de pricing opérationnel là où les systèmes industriels sont trop lourds ou indisponibles.
Stack

Compétences techniques — 6 technologies

Chaque technologie joue un rôle précis dans la chaîne de valeur quant.

📗

Excel VBA

Front-end du blotter, deal ticket options, dashboard Greeks en temps réel, génération de rapports. OOP via Class Modules, Dictionary, ListObjects. Le "cockpit" de l'outil vu par le trader et le risk manager.

🔵

VB.net

Bibliothèque de pricing interopérable avec Excel via COM Interop. Interface naturelle pour les quants issus du monde VBA. Implémentation Black-Scholes, arbres binomiaux, gestion des surfaces de volatilité.

🔷

C#

Moteur de pricing haute performance. Calcul des Greeks (Delta, Gamma, Vega, Theta, Rho), Monte Carlo parallélisé (Parallel.For), API REST pour exposer les services de pricing. QuantLib.NET ou implémentation custom.

🗄️

Access + SQL

Persistance des positions, historique des valorisations, tables de référence (underlyings, vol surface). Access pour les environnements standalone, SQL Server pour la production. ADO.NET pour l'accès depuis .NET.

🐍

Python

Analytics avancés : Monte Carlo VaR (NumPy/SciPy), backtesting des modèles, calibration des surfaces de vol (scipy.optimize), stress testing FRTB. Pandas pour l'analyse de séries temporelles historiques. Interop via xlwings ou API.

📐

Modèles Quant

Black-Scholes (closed-form), Arbres binomiaux (options américaines), Monte Carlo (path-dependent), Dupire Local Vol. Greeks analytiques et par différences finies. Calibration sur données de marché réelles.
