Toutes les tables sont sur SQL Server. Les tables existent déjà — on les réutilise. Microsoft Access est décommissionné.

💬
Pourquoi SQL Server et non Access ?
SQL Server c'est le champion poids lourd. Access c'était le débutant pratique. En salle de marché, tu peux avoir 50 traders qui lisent et écrivent des données en même temps. Access plante dans cette situation. SQL Server gère ça les yeux fermés.

🎓 Junior, imagine ça : Access, c'est comme un cahier de notes partagé entre 50 personnes. Si tout le monde écrit en même temps, c'est le chaos. SQL Server, c'est comme Google Docs : 1000 personnes peuvent modifier en même temps, et chaque modification est tracée, sécurisée, et annulable. En finance, ça n'a pas de prix.

Les tables existantes sur BFASERVER01 sont réutilisées directement. Le schéma est identique à ce qui était prévu pour la production.

💡 Expert Finance parle au Junior (comme si t'avais 5 ans) :
Tu vois la colonne "Acteur métier" ? C'est important de savoir QUI utilise QUOI et POURQUOI :
• 🧑‍💼 Trader : il veut saisir des deals et voir ses positions. Il n'aime pas les bugs. Il veut tout en temps réel.
• ⚠️ Risk Manager : il surveille que le Trader ne prend pas trop de risques. Il peut bloquer une position si la VaR dépasse la limite.
• 💻 Quant Dev : il construit l'outil. Il touche à tout mais surtout aux formules et aux données de marché.
• 🏛️ Compliance : il doit prouver que tout est tracé et conforme à la réglementation (MiFID2, FRTB).
